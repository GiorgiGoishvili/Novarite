# MVP Scope

## Goal

The goal of the Novarite MVP is to demonstrate the smallest working version of the platform that proves the core idea.

Novarite should not try to become a full Steam competitor in the first version. Instead, it should show a focused prototype of an indie game launchpad powered by Solana.

## MVP Features

The first version will include:

- Landing page
- Game marketplace prototype
- Creator dashboard prototype
- Developer profile concept
- Game listing concept
- Wallet connection placeholder
- Access pass purchase concept
- Player reward claim concept

## Not Included in MVP

The MVP will not include:

- Full game launcher
- Real game downloads
- Full production marketplace
- Complex trading system
- Advanced analytics
- Full multiplayer community tools
- Production payment processing

## Main User Flow

1. A visitor opens Novarite.
2. They learn what the platform does.
3. They browse indie game listings.
4. A developer can view the creator dashboard.
5. A wallet connection is shown as the ownership layer.
6. A player can simulate buying a game access pass.
7. A player can simulate claiming a reward.

## Success Criteria

The MVP is successful if it clearly shows:

- What Novarite is
- Who it is for
- Why Solana is useful
- How developers publish games
- How players receive ownership or rewards
- What future version can become
